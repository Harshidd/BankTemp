# Workflow & Ops SaaS Structural Architecture

## Layout System
- **Framework:** Next.js 16 (App Router + React 19)
- **Styling:** Tailwind CSS v4 (White/Slate-50 base)
- **Routing:** State-based internal router in `page.tsx`.

## Page Components
- `pages/Home.tsx`: Action-oriented hero and chaos-to-control framing.
- `pages/Features.tsx`: Technical capability matrix (Control, Visibility, Handoffs).
- `pages/Workflows.tsx`: Sectoral use-case library (RevOps, EngOps, Compliance).
- `pages/Pricing.tsx`: Tiered usage-based execution packages.
- `pages/Contact.tsx`: Formal demo request and technical support hub.
- `pages/FAQ.tsx`: Operational protocol hub with technical Q&A.

## Global Navigation Hubs
- `components/Navbar.tsx`: Fixed, backdrop-blur with amber state switching.
- `components/Footer.tsx`: Light industrial link directories with operational status indicators.

## Shared Visual Accents (Workflow DNA)
- Unified high-density grid background (low opacity).
- Bold italicized headings with amber underline motifs.
- Process-oriented paneling and checklist-driven feature blocks.

## Performance Logic
- Sub-ms state switching for zero-latency process visualization.
- High-velocity amber hover interactions.
- Optimized for high-density structural layouts.
